Chapter 4 Development

4.1 Repository Layout

Important paths in the repository:

  • R/: R wrapper functions exported by the package.
  • srcjs/: TypeScript and JavaScript source code for the auto-coder bridge.
  • inst/index.js: bundled JavaScript shipped with the R package.
  • tests/testthat/: package tests and test fixtures.
  • man/: generated R documentation.
  • docs/: rendered bookdown site.

4.2 Updating Documentation

Function documentation is written in roxygen comments in the R/ files. After editing those comments, regenerate the manual pages and namespace:

devtools::document()

The bookdown pages are written in the top-level .Rmd files listed in _bookdown.yml. Render the book after editing pages:

bookdown::render_book("index.Rmd")

4.3 Testing

Run the package tests after changes to the R wrappers or JavaScript bundle:

devtools::test()

The tests include examples for:

  • coding one response object,
  • coding arrays of response objects,
  • applying manual coding data, and
  • inspecting dependency trees.

4.4 Publishing With GitHub Pages

This book is configured to render into docs/. In the GitHub repository settings, set Pages to publish from the docs/ folder on the main branch.
